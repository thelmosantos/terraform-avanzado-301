## Qué aprenderás

- Trabajar Terraform en **flujo colaborativo**: ramas, Pull Requests y revisión de código.
- Distinguir conflictos de **código** de conflictos de **estado**.
- Buenas prácticas de revisión de cambios de infraestructura.

## Contexto

- En equipo, varias personas tocan el mismo código: Git evita pisarse.
- El **código** se mezcla en Git; el **estado** NO se resuelve a mano (lo veremos en M07/M08).
- El PR es la puerta con revisor: nada llega a `main` sin pasar por ahí.
