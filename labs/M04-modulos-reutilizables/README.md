## Qué aprenderás

- Qué es un módulo y cuándo conviene crearlo.
- Diseñar módulos con **inputs** (variables) y **outputs** bien definidos.
- Encapsular lógica reutilizable: naming corporativo, etiquetado y un S3 parametrizable.

## Contexto

- Si copias y pegas el mismo bloque, necesitas un módulo.
- Un módulo es como una **función**: entradas → recurso → salidas.
- Empieza con pocos inputs y crece según haga falta; menos es más.
